# Sweet Home Budget (우리 집 가계부) 개발 가이드

이 문서는 부부가 함께 사용하는 가계부 앱 `sweet-home-budget` 프로젝트의 구조와 개발 규칙을 정리한 문서입니다.

## 1. 프로젝트 개요
- **목적**: 부부간 실시간 지출/수입 공유 및 자산 관리.
- **기술 스택**: React (CRA), Vanilla CSS, Firebase (Auth & Firestore), Recharts (통계), XLSX (엑셀).
- **주요 특징**: PWA 지원, 투자 누적 관리, 할부/반복 결제 지원, 사용자 승인 시스템.

## 2. 핵심 기술 명세
### Firebase 연동 및 보안
- **Authentication**: Google 로그인을 사용하며, `adsl5964@gmail.com`은 시스템 관리자(Admin)로 하드코딩되어 즉시 접근 가능함.
- **사용자 승인 시스템**: 신규 유저는 로그인 시 `PendingApproval` 화면으로 이동하며, 관리자가 `users` 컬렉션에서 `approved: true`로 변경해야 앱 사용 가능.
- **Firestore 구조**: 
    - `transactions`: 수입/지출/투자 내역. `isWithdrawal` 필드로 투자 입/출금 구분.
    - `users`: 사용자 프로필 및 권한 상태 (`approved`, `isAdmin`).
    - `categories`: `type` 필드로 [지출, 수입, 투자(입금), 투자(출금)] 구분.

## 3. 핵심 비즈니스 로직
### 투자 관리 (Investment)
- **누적 합계**: 홈 화면과 통계 페이지의 '투자' 금액은 이번 달 내역만이 아닌, **과거부터 현재 달 말일까지의 누적액(입금 - 출금)**을 표시함.
- **카테고리 분리**: `investment_deposit`과 `investment_withdrawal`로 카테고리를 분리 관리하여 입력 상황에 맞게 필터링함.
- **반복 등록 (Recurring)**: 지출과 동일하게 투자 내역도 '매월 반복' 설정을 지원함. 체크 시 시작일로부터 향후 36개월간 매달 동일한 투자 내역을 자동 생성함.

### 지출 기능 (Expenses)
- **할부 (Installments)**: 최대 24개월 지원. 입력 시 선택한 개월 수만큼 자동으로 가상 내역을 생성하여 매달 분산 표기함.
- **반복 결제 (Recurring)**: 체크 시 시작일로부터 향후 36개월간 매달 동일한 내역을 자동 생성함.
- **데이터 확장**: `expandedTransactions` 유틸리티를 통해 원본 데이터를 UI용 가상 데이터로 변환하여 사용함.

### 통계 및 내보내기
- **추세 그래프**: 최근 6개월/12개월 단위로 카테고리별 소비 추세를 시각화함.
- **리스트 연동**: '항목별 상세 내역'의 카테고리 클릭 시 하단 추세 그래프가 해당 항목으로 즉시 필터링됨.
- **엑셀 추출**: `지출/수입/투자 요약` 시트와 `전체 상세 내역` 시트로 구성된 다중 시트 엑셀 파일을 생성함.

## 4. UI/UX 디자인 및 반응형 규칙
- **반응형 대시보드**: 좁은 화면(`360px` 이하)에서 폰트 크기 및 간격을 자동으로 축소하여 정보 겹침 방지.
- **내부 스크롤**: 홈 화면에서 달력은 고정되고 하단 내역 리스트만 개별 스크롤됨. 네비게이션 바에 가려지지 않도록 충분한 하단 패딩 확보.
- **색상 체계**: 수입(`#22c55e`), 지출(`#ef4444`), 투자(입금: `#f59e0b`, 출금: `#92400e`).
- **모바일 최적화**: 모든 입력 필드(`input type="date"` 포함)는 `width: 100%` 및 `box-sizing: border-box`를 적용하여 팝업 밖으로 나가지 않도록 함.

## 5. 배포 가이드 (Vercel)
- **Git 저장소**: `Sweet_Home_budget` 독립 레포지토리를 사용함.
- **빌드 설정**: `CI=false npm run build` 명령어를 사용하여 ESLint 경고로 인한 빌드 중단을 방지함.
- **보안**: Vercel 도메인을 Firebase 콘솔의 '승인된 도메인'에 반드시 추가해야 구글 로그인이 작동함.

## 6. 개발 시 주의사항
1. **데이터 확장성**: `expandedTransactions` 로직 수정 시 원본 ID(`originalId`)를 보존하여 수정/삭제가 원본에 반영되도록 해야 함.
2. **잔액 계산**: '잔액'은 순수하게 **(이번 달 수입 - 이번 달 지출)**로 계산하며, 투자는 포함하지 않음 (사용자 요청 사항).
3. **PWA 스플래시**: `index.html` 내의 `.initial-loading` 스타일과 `manifest.json`의 배경색을 일치시켜 부드러운 앱 실행 환경 유지.
